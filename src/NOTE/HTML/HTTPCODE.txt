상태 코드
1xx : 요청 처리중 (거의 사용하지 않음)
2xx : 정상 처리 완료(성공)
3xx : 완료하려면 추가 행동 필요
4xx : 클라이언트 오류, 잘못된 문법
5xx : 서버 오류, db가 내려가거나 서버가 정상 동작을 못할떄

200 : OK 요청 성공
201 : 요청 성공해서 새로운 리소스 생성
204 : No Content
->서버가 요청을 성공적으로 수행했지만 본문에 보낼 데이터가 없음

301, 308 : 리소스의 URI가 영구적으로 이동 301은 본문이 제거될수 있고 308은 본문 유지

302,307,303(일시적 리다이렉션) 일시적으로 URI가 변경되어서 URL를 변경하면안됨
302: 본문이 제거 될 수도 있고 안될수도있음(대부분 변함), (현재 대부분 사용)
307: 요청 메서드와 본문 유지
303: 302와 기능은 같으나 리다이렉션시 요청 메서드가 GET으로 변함

PRG:POST/Redirect/Get
POST로 요청후 브라우저를 새로고침하면? 중복 요청이 될수있음
처음 요청을하고 응답을 302번으로 줌(GET으로 바꿈) 다음 새로고침시 POST가 아닌 GET으로 요청

300,304:기타 리다이렉션
300 : 안씀
304 : 캐시를 목적으로 사용, 클라이언트에게 리소스가 수정되지 않았음을 알려줌
304는 로컬 캐시를 사용해야 하므로 메시지 바디를 포함하면 안됨됨

400대 오류는 클라이언트에 오류가 있기때문에 여러번 시도를 해도 똑같은 오류가 발생함(수정을 해서 재요청해야함)
그러나 500대 오류는 서버에 오류가 있기떄문에 재시도하면 성공할수도 있음
400 : 요청 구문, 메시지 오류 등등 파라미터가 잘못되거나 API 스펙이 맞지 않을 떄
401 : 인증이 되지 않음
403 : 로그인은 되나 등급이 충분 하지 않은 경우
404 : 요청 리소스가 서버에 없음 URL이 잘못된 경우

500 : 서버 내부 문제
503 : 서버가 일시적인 과부화
